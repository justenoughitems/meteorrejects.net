What it is

Everything Meteor Left Behind

Meteor Rejects collects features that got cut, rejected, or never officially shipped — and gives them a permanent home.

Restored Modules

Features once in Meteor Client but removed over time — AimAssist, InstaMine, AntiBot, and more — fully restored and kept up to date.

Unmerged Pull Requests

Community PRs that were never merged into the main Meteor repo — AutoPot, PacketFly, TreeAura, and dozens more — all live here.

Client Ports

Best-in-class features cleanly ported from Wurst, BleachHack, LiquidBounce, JexClient, and Cornos into Meteor's architecture.

Drop-In Compatible

Getting started

Install in 60 Seconds

Four simple steps and you're running 58+ extra modules.

Steps

1
Install Meteor Client
Get the latest dev build from meteorclient.com for your version.
2
Download Meteor Rejects
Grab the .jar from the Download section matching your Minecraft version.
3
Drop it in mods
Place the .jar in .minecraft/mods alongside Meteor Client.
4
Launch with Fabric
Start Minecraft via Fabric Loader 0.15+. All modules appear in Meteor's GUI instantly.

Requirements

Minecraft Java Edition — 1.21.0, 1.21.4, or 1.21.11
Fabric Loader 0.15 or newer
Meteor Client (dev build recommended)
Java 17 or newer

Only compatible with Minecraft Java Edition. Bedrock Edition is not supported. Verify all builds target the same Minecraft version.

FAQ

Frequently Asked Questions

Everything you need to know before installing.

Meteor Rejects is a free, open-source addon for Meteor Client that restores features which were rejected in pull requests, removed from the main codebase, or ported from clients like Wurst, BleachHack, LiquidBounce, and JexClient. It adds 58+ modules and 14+ commands unavailable in the base Meteor Client.

First install Meteor Client, then download the Meteor Rejects .jar for your Minecraft version. Drop it into your .minecraft/mods folder and launch with Fabric Loader 0.15+. All new modules and commands appear automatically in the Meteor Client interface.

Meteor Rejects supports Minecraft Java Edition 1.21.0, 1.21.4, and 1.21.11 — with a separate .jar for each. Bedrock Edition is not supported. Always download the version that matches your Minecraft install.

Yes. Meteor Rejects is fully free and MIT licensed. The complete source code is on GitHub at github.com/AntiCope/meteor-rejects for anyone to review, fork, or contribute to. There are no premium tiers, paywalls, or ads.

Meteor Client is the base mod — it must be installed first. Meteor Rejects is an addon that extends it. Meteor Client focuses on well-curated features while Meteor Rejects specifically collects everything that was removed, rejected, or never officially included. They are fully compatible and work together seamlessly.

Meteor Rejects includes a built-in Duplicate Module Names option to prevent naming conflicts with other addons. In general it is compatible with most Meteor addons — just ensure all addons target the same Minecraft and Meteor Client version.

Fabric Loader 0.15 or newer is required. Download the latest version from fabricmc.net. If Meteor Client is already working on your setup, your Fabric Loader version is almost certainly compatible.

Open an issue on GitHub at github.com/AntiCope/meteor-rejects/issues. Include your Minecraft version, Meteor Client version, and steps to reproduce when filing a bug report. Feature requests are also welcome via the Issues tab.

Absolutely. Fork the repo on GitHub, make your changes, and open a pull request. The project welcomes ports from other clients, restorations of removed Meteor features, and bug fixes for existing modules. See the contributing guide in the repository for details.
